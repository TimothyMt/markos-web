# Slice P0.1 — Strategy Spine (4 neo) — intake + lưu trữ

> **Mục tiêu:** cho user khai **xương sống chiến lược** mà mọi campaign/bài sẽ bám: ① Objective đo được **bằng SỐ** · ② Audience/ICP · ③ Định vị · ④ Capacity.
> Slice này CHỈ làm intake + lưu, chưa bơm vào prompt (đó là P0.2).
> **Đọc trước:** `docs/cmo/00-PLAN.md` mục "6 NGUYÊN TẮC THIẾT KẾ" — nguyên tắc 1,2,3,6 áp thẳng vào slice này.

## Vì sao 4 neo (không chỉ objective)
- Objective một mình **không** quyết định kênh hay thông điệp — *audience* + *định vị* mới quyết định. Thiếu 2 neo này → hệ thống sản xuất hiệu quả nội dung **không khác biệt**.
- Metric/target/baseline phải **là số** thì P4 (vòng đo-học) mới so được gap-vs-target. Chuỗi tự do = vòng lặp không đóng.
- *Capacity* là biến để các phase sau **cắt phạm vi** (số kênh, độ dài journey). Persona thường 1 người → làm ít mà sâu.

## Dữ liệu — key mới trong `intake_extra` (KHÔNG đổi schema)
```
intake_extra.spine = {
  "objective": {
    "outcome":  str,          # kết quả muốn (vd "tăng đơn online", "ra mắt SP mới", "gom lead tư vấn")
    "metric":   str,          # nhãn chỉ số (vd "số đơn", "lead", "doanh thu")
    "target":   { "value": number|null, "unit": str, "period": str },  # đích (vd 300, "đơn", "tháng")
    "baseline": { "value": number|null, "unit": str, "period": str },  # hiện tại (vd 120, "đơn", "tháng")
    "deadline": str           # mốc (vd "trước Tết 2026")
  },
  "audience": {
    "who":   str,             # tệp chính / ICP (vd "mẹ bỉm 25-35, TP.HCM")
    "pain":  str,             # nỗi đau / nhu cầu cốt lõi
    "where": str              # họ hay ở đâu (gợi ý kênh, vd "TikTok, group FB")
  },
  "positioning": str,         # 1 CÂU "vì sao chọn mình thay vì đối thủ"
  "constraint": {
    "people":   str,          # nhân lực (vd "1 người tự làm")
    "budget":   str,          # ngân sách ads (vd "~3tr/tháng")
    "capacity": str           # nhịp làm nổi (vd "3 bài/tuần") — neo LÁI phạm vi P2/P3
  }
}
```
**Cho phép để trống mọi field** — lưu chuỗi rỗng / `value: null`, đừng bắt buộc. `value` khi có PHẢI parse ra số (nhận cả "300", "300 đơn", "~120" → lấy phần số; không parse được → null). `unit`/`period` giữ text.

## Cấu trúc trường — đối chiếu chuẩn EVAL
Cấu trúc 4 neo trên là **đề xuất khởi điểm**, chưa phải chốt. Trước khi chốt `audience`/`positioning`, lướt skill `product-marketing` (`marketingskills`) như **kho ứng viên trường** (không phải trọng tài) → mọi thêm/bỏ trường phải qua **`docs/cmo/EVAL.md`** (Test 1 Consumption / Test 2 Answerability theo bậc / Test 3 Empirical).
- Ở P0.1: chỉ cần qua **Test 1 + Test 2** (Test 3 cần chạy gen → hoãn P0.2).
- Mọi **lệch** cấu trúc → ghi lý do theo EVAL trong commit. Không bê nguyên framework, không đổi mù.

### F1 — `save_spine()` (backend ghi) · `webapp/business.py`
- Grep `save_messaging` / `save_pillars` (KHÔNG đọc cả file) → theo khuôn.
- `async def save_spine(user_id=None, spine: dict = None) -> dict:` — load profile → set `intake_extra["spine"] = {...}` (chỉ giữ đúng cấu trúc 4 neo, strip, cắt độ dài hợp lý, **ép `value`→ số/None**) → `save_profile`/store update → `{"ok": True}`.

### F2 — expose `bizSpine` · `webapp/business.py biz_data()`
- Trong `biz_data()`: thêm `bizSpine = intake_extra.get("spine", {})`.
- Trivial → **có thể gộp commit với F1** nếu cùng phiên; nếu tách: `feat(cmo): P0.1 F2 expose bizSpine`.

### F3 — route API · `webapp/api.py`
- Handler `biz_spine_save`: đọc JSON `{spine:{...}}` → `business.save_spine`. Đăng ký `Route("/api/biz/spine/save", ...)` trong `api_routes()`. Theo khuôn route messaging/pillars.
- Commit: `feat(cmo): P0.1 F3 route /api/biz/spine/save`.

### F4 — FE khối "🎯 Chiến lược nền" · `web/app.js` **+ mirror standalone (CÙNG 1 COMMIT)**
- Khối ở đầu trang Chiến lược (nêu lý do chỗ đặt trong commit). 4 nhóm ô:
  - **Mục tiêu:** outcome · metric · target (số+đơn vị+kỳ) · baseline (số+đơn vị+kỳ) · deadline.
  - **Tệp khách:** who · pain · where.  **Định vị:** 1 ô.  **Năng lực:** people · budget · capacity.
- Nút Lưu → POST `/api/biz/spine/save`; render lại từ `bizSpine`.
- **Mirror y hệt sang `dashboard-standalone.html`** (`<script>` + `<style>`) — **trong cùng commit này** (không để 2 bản lệch).
- Commit: `feat(cmo): P0.1 F4 FE khối Chiến lược nền + mirror standalone`.
