## Vì sao cấu trúc này (đã validate 2026-07, không phán solo)
- **Positioning KHÔNG phải 1 câu tự do.** Theo Dunford, định vị là **quan hệ**: bắt đầu từ *khách làm gì nếu không có bạn* (lựa chọn thay thế) → *bạn có gì mà cái đó không có* (khác biệt). Thiếu khung này, định vị "nghe hay nội bộ nhưng vô dụng với khách vì không khác biệt" ([nguồn](https://www.aprildunford.com/post/a-quickstart-guide-to-positioning)). → lưu tối thiểu `alternative` + `differentiator`, không phải 1 chuỗi.
- **Metric/target/baseline là SỐ** thì D6 (đo-học) mới so gap-vs-target.
- **Capacity = ràng buộc** để miền sau **cắt phạm vi** (số kênh D3, độ dài journey D4). Giữ nguyên key `constraint`.

## Dữ liệu — key mới trong `intake_extra` (KHÔNG đổi schema)
```
intake_extra.spine = {
  "objective": {
    "outcome":  str,          # kết quả muốn (vd "tăng đơn online", "gom lead tư vấn")
    "metric":   str,          # nhãn chỉ số (vd "số đơn", "lead", "doanh thu")
    "target":   { "value": number|null, "unit": str, "period": str },  # đích (vd 300, "đơn", "tháng")
    "baseline": { "value": number|null, "unit": str, "period": str },  # hiện tại (vd 120, "đơn", "tháng")
    "deadline": str           # mốc (vd "trước Tết 2026")
  },
  "audience": {
    "who":   str,             # tệp chính / ICP (vd "mẹ bỉm 25-35, TP.HCM")
    "pain":  str,             # nỗi đau / nhu cầu cốt lõi
    "where": str              # họ hay ở đâu (gợi ý kênh)
  },
  "positioning": {            # ★ cấu trúc quan hệ (Dunford), KHÔNG còn là 1 chuỗi
    "alternative":   str,     # khách làm gì nếu không có bạn (đối thủ / cách thay thế)
    "differentiator":str,     # bạn có gì mà cái đó KHÔNG có
    "statement":     str      # 1 câu định vị tổng (tuỳ chọn — user tự viết hoặc để trống)
  },
  "constraint": {             # neo RÀNG BUỘC (lái phạm vi D3/D4), không phải trụ chiến lược
    "people":   str,          # nhân lực (vd "1 người tự làm")
    "budget":   str,          # ngân sách ads (vd "~3tr/tháng")
    "capacity": str           # nhịp làm nổi (vd "3 bài/tuần")
  }
}
```
**Cho phép để trống mọi field** — chuỗi rỗng / `value: null`. `value` khi có PHẢI ép ra số (xem F1 — **locale Việt**). `unit`/`period`/text giữ nguyên (strip + cắt độ dài).

> Theo `docs/cmo/WORKFLOW.md`. Tuần tự F1→F4, mỗi F chờ review PASS mới sang F kế.

### F1 — `save_spine()` (backend ghi) · `webapp/business.py`
> **Lưu ý:** đã có bản F1 nháp (uncommitted) dùng positioning=chuỗi + parse số locale Tây. **Làm lại F1** cho khớp brief đã vá: positioning = **object**, và ép số theo **locale Việt**.
- Grep `save_messaging` / `save_pillars` (KHÔNG đọc cả file) → theo khuôn.
- `async def save_spine(user_id=None, spine: dict = None) -> dict:` — load profile → set `intake_extra["spine"] = {...}` (giữ đúng cấu trúc trên: objective/audience/**positioning object**/constraint; strip; cắt độ dài; **ép `target/baseline.value` → số/None**) → store update → `{"ok": True}`.
- **Ép số theo QUY ƯỚC SỐ VIỆT NAM** (`,` = thập phân, `.` = phân nhóm nghìn):
  - Bỏ ký tự không phải chữ số/`.`/`,`/`-`; chỉ lấy cụm số đầu.
  - Có cả `.` và `,`: dấu **cuối cùng** là thập phân, các dấu trước là phân nhóm.
  - Chỉ có `.` → phân nhóm nghìn, bỏ hết (`1.500`→1500, `1.500.000`→1500000).
  - Chỉ có `,` → thập phân (`1,5`→1.5).
  - Parse fail → `None`. Số thật (int/float) → `float`.
  - Test khô bắt buộc (dán vào commit): `1.500→1500.0 · 1.500.000→1500000.0 · 1,5→1.5 · 2,5 tỷ→2.5 · 300 đơn→300.0 · abc→None`.
- Commit: `feat(cmo): P0.1 F1 save_spine (positioning object + số locale VN)`.

- Commit: `feat(cmo): P0.1 F3 route /api/biz/spine/save`.

### F4 — FE khối "🎯 Chiến lược nền" · `web/app.js` **+ mirror standalone (CÙNG 1 COMMIT)**
- Khối đầu trang Chiến lược. 4 nhóm ô:
  - **Mục tiêu:** outcome · metric · target (số+đơn vị+kỳ) · baseline (số+đơn vị+kỳ) · deadline.
  - **Tệp khách:** who · pain · where.
  - **Định vị (2 ô + 1 tuỳ chọn):** `alternative` ("Không có bạn, khách làm gì?") · `differentiator` ("Bạn có gì họ không có?") · `statement` (câu định vị, tuỳ chọn).
  - **Năng lực:** people · budget · capacity.
- Nút Lưu → POST `/api/biz/spine/save`; render lại từ `bizSpine`.
- **Mirror y hệt sang `dashboard-standalone.html`** (`<script>` + `<style>`) — **cùng commit**.
- Commit: `feat(cmo): P0.1 F4 FE khối Chiến lược nền + mirror standalone`.
