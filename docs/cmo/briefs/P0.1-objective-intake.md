# Slice P0.1 — Objective intake + lưu trữ

> **Mục tiêu:** cho user khai 1 **MỤC TIÊU business đo được** — xương sống mà mọi chiến lược/campaign/bài sẽ bám. Slice này CHỈ làm intake + lưu, chưa bơm vào prompt (đó là P0.2).

## Dữ liệu — key mới trong `intake_extra` (KHÔNG đổi schema)
```
intake_extra.objective = {
  "outcome":   str,   # kết quả muốn (vd "tăng đơn online", "ra mắt SP mới", "gom lead tư vấn")
  "metric":    str,   # đo bằng gì (vd "số đơn/tháng", "lead/tuần", "doanh thu")
  "target":    str,   # đích (vd "300 đơn/tháng")  — để string, KHÔNG ép số
  "baseline":  str,   # hiện tại (vd "~120 đơn/tháng")
  "deadline":  str,   # mốc (vd "trước Tết 2026")
  "constraint":str    # ràng buộc (ngân sách/nhân lực, vd "1 người làm, ~3tr ads/tháng")
}
```
Cho phép để trống field — lưu chuỗi rỗng, đừng bắt buộc hết.

- Thêm `async def save_objective(user_id=None, objective: dict = None) -> dict:` — theo khuôn `save_messaging` (line ~2733) / `save_pillars` (~339): load profile → set `intake_extra["objective"] = {...}` (chỉ giữ 6 key trên, strip, cắt độ dài hợp lý) → `save_profile`/store update → trả `{"ok": True}`.
- Trong `biz_data()` (line ~166): expose ra key `bizObjective` = `intake_extra.get("objective", {})` để FE đọc.
