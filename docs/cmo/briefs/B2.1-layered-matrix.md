## 🧱 Mô hình dữ liệu (2 lớp — key trong `intake_extra`, KHÔNG đổi schema DB)

### Lớp MA TRẬN NỘI DUNG — `intake_extra.content_matrix` (MỚI, thường trực, độc lập đợt)
Ma trận xương sống: **trụ × phễu**, mỗi ô mang **nền tảng** (repurpose) + nhịp gợi ý. Đây là "cái chạy đều".
```
intake_extra.content_matrix = {
  "cells": [                                  # trụ × phễu — CHỈ ô có nội dung (không ép đủ 100% ô, luật volume)
    {
      "pillar": "<trụ = messaging.pillars[].territory>",   # đa ngành: sinh từ T3, KHÔNG hardcode
      "tier": "tofu|mofu|bofu",                            # phễu chạy TRONG trụ
      "role": "<ô này làm gì trong phễu của trụ — 1 câu>",
      "platforms": ["<nền tảng hợp archetype/current_channels>"],  # 1 ô repurpose sang các nền tảng này
      "cadence": "<nhịp gợi ý: vd '1 bài/2 tuần' | '' = tuỳ chọn>"
    }
  ],
  "updated_at": <ts>
}
```

### Lớp ĐỢT NHẤN — `intake_extra.key_ideas[]` (nâng B2, thêm trục nhấn + repurpose)
Đợt nhấn = **cao điểm**: đẩy MỘT tầng phễu (hoặc vài trụ) mạnh lên trong 1 kỳ hạn. Field THÊM (additive):
```
key_idea += {
  "focus_tier": "tofu|mofu|bofu|",     # đợt ĐẨY tầng nào across trụ. '' = suy từ goal (B2 đã map goal→ratio)
  "focus_pillars": ["<trụ A>", ...]    # đợt tập trung trụ nào. [] = trải mọi trụ
}
key_idea.funnel_map.posts[] += {       # mỗi bài giờ có toạ độ ma trận + nhóm repurpose
  "pillar": "<trụ — ô nào>",           # '' nếu bài cũ/không xác định trụ (degrade: coi như trụ chung)
  "sibling_group": "<group-id | ''>"   # cùng group = 1 nội dung repurpose sang nhiều nền tảng (nền tảng axis)
}
# post đầy đủ = {pillar, tier, channel, role, note, sibling_group}   — 4 khoá cũ GIỮ NGUYÊN
```

## ① `gen_content_matrix` — dựng MA TRẬN NỘI DUNG (trụ × phễu × nền tảng) (1 commit) — MỚI
**File:** `webapp/business.py` (hàm mới, cạnh `gen_funnel_map_for_idea`).
**ĐỌC:** `messaging.pillars` (trụ) · `playbook_struct` (tier×channels, degrade) · `synthesis` · archetype · `current_channels`.
**GHI:** `intake_extra.content_matrix`.

- Trụ = `messaging.pillars[].{territory,angle}` (đa ngành — producer `save_messaging` ✓). Thiếu pillars → degrade:
  1 "trụ chung" từ `messaging.core`/synthesis. Thiếu cả → `{error}` mềm ("chưa có Thông điệp để dựng ma trận").
- Với mỗi trụ, xoay qua 3 tầng phễu (tofu/mofu/bofu), lấy **kênh gợi ý** từ `playbook_struct` Hướng khớp trụ
  (`channels`) → degrade → chọn nền tảng theo **archetype** (luật đa ngành #2) giao với `current_channels`.
- 1 LLM call JSON → `{cells:[{pillar,tier,role,platforms,cadence}]}`. **Volume tỉnh táo (luật #5 + handoff 9 bài/tuần
  quá tải team 2-3):** KHÔNG ép đủ mọi ô trụ×tier; chọn ô đắt giá, nhịp thưa hợp nguồn lực.
- **Validate (chống lỗi funnel cũ):** `tier ∈ {tofu,mofu,bofu}` (rác → bỏ ô) · `pillar` phải thuộc danh sách trụ
  đã biết (khớp lỏng, không khớp → gán trụ gần nhất / bỏ) · `platforms` cắt list, rỗng → điền 1 nền tảng archetype ·
  `role` bắt buộc (thiếu → bỏ ô). Trần token cứng, KHÔNG lưu string đứt.
- **Degrade:** cells rỗng sau validate → dựng tối thiểu 1 ô/trụ ở tofu (kênh archetype). KHÔNG trả cụt.
- **Derived-state? KHÔNG** — content_matrix là *gợi ý cấu trúc* non-binding, user sửa/xoá ô = override sẵn trong thiết kế.
- Ghi `content_matrix` + `updated_at`. Trả `{ok, content_matrix}`.
